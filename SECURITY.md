# Security Policy

## Supported Versions

Only the latest release is actively maintained with security fixes.
Older versions may receive critical patches on a best-effort basis.

## Reporting a Vulnerability

**Do not file a public GitHub issue for security vulnerabilities.**

Please report vulnerabilities privately via GitHub's **Private vulnerability reporting** tool:

1. Go to the repository's [Security tab](https://github.com/upcloud-tools/upcloud-csi/security)
2. Click **Report a vulnerability**
3. Fill in the details

You can expect an acknowledgment within 72 hours, and we'll work with you
to understand the scope and impact. Once a fix is ready, we'll publish a
security advisory and credit the reporter (unless anonymity is requested).

For sensitive or critical issues, you may also email the maintainers directly —
reach out via the reporting form to request contact details.
